Standup Timer
Run focused daily standups with per-person countdowns, sound alerts, and a session summary.
Open tool →
What it does
Gives every team member their own countdown timer during standups. An animated ring fills as time is used, shifting from blue → amber (last 30s) → red (over time). When one person finishes, the next starts automatically. At the end you get a full session summary showing who used how much time.
When to use it
✓Daily team standups
✓Sprint review introductions
✓Keeping remote meetings on time
✓Round-robin updates in any meeting
How to use
1Set default time per person (e.g. 2:00) in the left panel
2Type each team member's name and press Enter or +
3Drag names to reorder the queue
4Click ▶ Start Standup — the first person's timer begins
5Click ✓ Done · Next to move on, or Skip to skip someone
💡Click any chip in the queue strip at the bottom to jump directly to that person mid-session.
🔔Enable Sound alerts in Options — you'll hear a warning beep at 10 seconds, a chime when time's up, and a fanfare when the session ends.
⏱Enable Allow overtime if speakers regularly run over — the timer goes red and negative instead of forcing an auto-advance.

Team Timezone Clock
See live clocks for every team member's city, with DST detection and working hours status.
Open tool →
What it does
Shows live analog + digital clocks for any city in the world. Searches by city name, state, country, or timezone abbreviation. A colour-coded indicator (🟢 working / 🟡 early-late / 🔴 off hours) tells you at a glance whether someone is reachable. DST is detected automatically and shown on each card.
When to use it
✓Scheduling meetings across time zones
✓Checking if a colleague is in working hours
✓Keeping it open on a second monitor during calls
✓Quickly answering "what time is it in Dallas?"
How to use
1Click + Add member
2Type a city (e.g. Dallas, Bengaluru, IST, PST)
3Select the city from the dropdown — it shows UTC offset and ☀️ DST if active
4Add an optional name (e.g. "Priya" or "NYC Office")
5All clocks are saved — they persist when you close and reopen the tab
💡Searching Dallas shows "Dallas, TX, USA" — and the card correctly shows Dallas with a note "Zone: Chicago, USA" so there's no confusion.
☀️The DST badge on each card lights up when that timezone is currently observing Daylight Saving Time — helpful when US and EU clocks shift at different times.

Checklist Maker
Turn any list into a structured, interactive checklist with sections, priorities and drag-to-reorder.
Open tool →
What it does
Converts raw lists into actionable checklists. Supports sections (group items under headers), priority flags, inline notes per item, drag-to-reorder within and across sections, progress tracking, and export to PDF or plain text.
When to use it
✓Sprint or release checklists
✓Onboarding steps for new team members
✓Meeting action items
✓Deployment runbooks
✓Any repeatable process checklist
How to use
1Option A: Paste a list in the left panel — lines starting with # become section headers
2Option B: Type one item at a time and press Enter
3Option C: Upload a .txt or .csv file
4Drag items between sections using the ⠿ handle on the left
5Click 🔥 for priority, or click the note field to add a description
6Click ⬇ Download PDF to print or save
💡Paste format: lines starting with # Setup create a dark section header. Items below it belong to that section until the next # heading.
📎Use Copy as text to get a [ ] / [x] formatted list you can paste into Slack, Notion, or a GitHub issue.

JSON Formatter
Format, validate, minify, sort, repair and visually explore any JSON.
Open tool →
What it does
Takes messy or minified JSON and makes it readable. Also validates it, shows parse errors, sorts keys alphabetically, minifies for production, and auto-repairs common mistakes (trailing commas, unquoted keys, single quotes, comments). The right panel shows a collapsible tree view with dot-notation path display and a search tab.
When to use it
✓Debugging API responses
✓Validating config files before deployment
✓Exploring deeply nested JSON structures
✓Minifying JSON for a request body
How to use
1Paste JSON into the Input panel — it formats automatically
2Use the indent selector to switch between 2 spaces / 4 spaces / tabs
3Click ⇅ Sort keys to alphabetise all keys recursively
4Click 🔧 Repair if you get a parse error — it fixes most common mistakes
5Click nodes in the Tree tab to see the full dot-notation path
6Use the Search tab to find any key or value
🔧Repair fixes: trailing commas like {a:1,}, unquoted keys like {name:"val"}, single-quoted strings, and // comments.
💡The stats bar shows file size, line count, total key count, and max nesting depth — useful for understanding complexity at a glance.

JSON Schema Validator
Validate JSON against a JSON Schema Draft-7 definition with line-numbered editors and detailed error cards.
Open tool →
What it does
Checks whether a JSON document conforms to a JSON Schema. Shows exactly which fields failed, what was expected vs. what was found, and the dot-notation path to each error. Both editors have line numbers, cursor position (Ln/Col), character count, and error line highlighting so you can find problems instantly.
When to use it
✓Validating API request/response shapes
✓Checking config files against a schema
✓Testing data pipelines produce valid output
✓Learning JSON Schema syntax interactively
How to use
1Paste your JSON data in the left editor
2Paste your JSON Schema in the right editor
3Click ▶ Validate — errors appear as cards with path + message
4Click Format above either editor to pretty-print it
5Try Sample: User or Sample: Product to see a working example
📍When a parse error occurs, the line number in the gutter turns red — click it or look at the Ln indicator at the bottom to navigate directly.
💡Supports full Draft-7: type, required, enum, pattern, format (email/uri/date/uuid/ipv4), allOf, anyOf, oneOf, not, if/then/else, $ref.

Config Diff
Compare two JSON, YAML, or INI config files structurally — not just line by line.
Open tool →
What it does
Parses both configs into objects and diffs them key-by-key. Shows Added (green), Removed (red), and Changed (amber with WAS → NOW values) as individual cards. Because it's structural, reordering keys or reformatting the file never creates false positives — only real value changes appear.
When to use it
✓Comparing dev vs prod config files
✓Reviewing what changed in a deployment
✓Auditing environment variable differences
✓Spotting missing keys between environments
How to use
1Paste Config A (original) on the left
2Paste Config B (modified) on the right
3Format is auto-detected (JSON / YAML / INI) and shown in the toolbar
4Diff appears instantly — Added / Removed / Changed cards on the right
5Click ⎘ Copy report to get a plain text summary
💡Try JSON sample, YAML sample, or INI sample in the topbar to see realistic dev/prod config differences immediately.
⇄Use the Swap button to flip A and B — useful when you want to see the diff from the opposite direction.

API Tester
Send real HTTP requests from the browser. Headers, auth, body, response viewer, history, cURL import/export.
Open tool →
What it does
A lightweight browser-based HTTP client. Send GET, POST, PUT, PATCH, DELETE requests to any public API. Add custom headers, query params, a request body, and authenticate with Bearer tokens, Basic Auth, or API keys. The response panel shows the status code, response time, body (JSON highlighted), and headers. Keeps a history of the last 50 requests.
When to use it
✓Quick-testing a public REST API
✓Verifying auth token formats
✓Checking response shapes before coding
✓Sharing a request as a cURL command
How to use
1Select a method (GET/POST…) and enter the URL
2Add Headers or Query Params in the tab below — toggle rows on/off with the checkbox
3For POST/PUT, paste your JSON in the Body tab and click Format
4Set auth in the Auth tab — Bearer, Basic, or API Key
5Press Enter in the URL bar or click Send ▶
⚠️If you see a CORS error, the server doesn't allow browser requests from other origins. This is a server-side restriction — not a bug. Test from Postman or add CORS headers to your server.
📋Use Export cURL to copy the request as a curl command you can share with teammates or run in a terminal.

Each field has a name and type (uuid, email, name, phone, date, url, status, lorem, city, array, etc.) and the generator produces realistic-looking values. Wrap the result in a single object, array, paginated response, or an envelope with meta and links.
When to use it

Regex Tester
Test, debug and explore regular expressions with live highlighting, flag toggles, and a built-in pattern library.
Open tool →
What it does
A live regex playground. Type a pattern and test text — matches highlight in colour-coded spans instantly. See every match as a card with position and capture groups. Test replacements with $1, $& syntax. Choose from 23 pre-built patterns (emails, URLs, JWTs, IPs, dates…) or build from the quick-reference panel.
When to use it
✓Writing and debugging regex patterns
✓Validating input format patterns
✓Building find-and-replace transforms
✓Learning regex syntax interactively
How to use
1Type your pattern between the /slashes/ at the top
2Paste test text in the Test text tab — matches highlight live
3Toggle flags (g, m, i, s, u) as needed
4Switch to Matches tab to see every match with position and groups
5Switch to Replace tab, type a replacement, and see the result live
💡Click any row in the Quick Reference panel to insert that token at the cursor. Great for learning — click \b to see it added and immediately see which words it matches.
📚The Pattern Library (left panel) has 23 common patterns — click one to load it instantly. Search by name like "email" or "JWT".
Keyboard shortcuts
/ Focus search g flag: find all matches i flag: case insensitive m flag: multiline ^ $

Cron Builder
Build cron expressions visually with dropdowns. See the next 5 scheduled run times instantly.
Open tool →
What it does
Visual cron expression builder with 5 dropdowns (minute, hour, day of month, month, day of week). Displays a plain-English description of the schedule, shows the next 5 run times, validates syntax, and lets you import/export cron expressions. Includes 5 common presets (Daily 9am, Weekly Monday, Monthly 1st, Every 5 minutes, Every 4 hours).
When to use it
✓Scheduling cron jobs without memorizing syntax
✓Planning recurring backups or maintenance tasks
✓Debugging cron schedules — see next run times instantly
✓Learning cron syntax interactively
How to use
1Click any Preset button to load a common schedule (Daily 9am, Weekly Mon, etc.)
2Or manually set each field using the dropdowns (minute, hour, day, month, day-of-week)
3See English description of the schedule update live
4Check Next 5 run times on the right to verify the schedule is correct
5Click ⎘ Copy to get the cron expression, then paste into your scheduler
💡Use ? (any) in day-of-month or day-of-week when you don't care about one — e.g. "every Tuesday" means day-of-month = ? and day-of-week = Tuesday.
⏰Times shown are based on your local timezone. Copy the cron and import it to your server's timezone as needed.
📋Paste any cron expression in the Import field to reverse-engineer it and see the next run times.

Secret Redactor
Automatically detect and redact passwords, API keys, tokens, emails, and more before sharing text.
Open tool →
What it does
Scans pasted text for 11 categories of sensitive data and replaces them with asterisks (★★★★★), black bars (█), or [REDACTED]. Works entirely in your browser — nothing is sent anywhere. Each category is togglable so you can keep emails visible but hide all API keys, for example.
When to use it
✓Sharing logs with teammates or in tickets
✓Sanitising config files before committing
✓Preparing screenshots or recordings
✓Creating safe examples from real data
How to use
1Paste any text into the left panel — secrets are highlighted immediately
2Toggle pattern categories in the left panel to enable/disable types
3Add custom keywords to catch company-specific secrets
4Choose redaction style: ★★★★★, █ bar, or [REDACTED]
5Click ⎘ Copy, ⬇ .txt, or ⬇ PDF to export
🔒Everything runs 100% in your browser. Your secrets never leave your machine — not even to this server (there isn't one).
💡Each category is colour-coded in the output: red = passwords, blue = API keys, green = emails, amber = tokens. This helps you verify redaction coverage at a glance.

Color Palette Manager
Pick colours, check WCAG contrast, generate palettes, and export to CSS/Tailwind/SCSS.
Open tool →
What it does
A full-featured colour tool for designers and front-end developers. Native colour picker synced with HEX input and RGB sliders. Checks WCAG AA/AAA contrast ratios against white and black. Generates palette variations (shades, tints, complementary, triadic, analogous, split-complementary). Saves a named palette and exports to CSS variables, Tailwind config, or SCSS.
When to use it
✓Building a design system colour palette
✓Checking text contrast for accessibility
✓Converting a HEX code to RGB or HSL
✓Generating a full tint/shade scale from a brand colour
✓Exporting colours for a Tailwind or CSS project
How to use
1Click the colour swatch to open the native picker, or type a HEX into the input
2Click any value box (HEX / RGB / HSL) to copy it instantly
3Check WCAG contrast against white and black in the Accessibility section
4Click a palette generator button (Shades, Tints, Triadic…) to add variations

Text Diff Checker
Compare two text snippets side-by-side or in unified view with word-level precision.
Open tool →
What it does
Uses the LCS (Longest Common Subsequence) algorithm to find the minimal set of changes between two texts. Shows added lines in green and removed lines in red. Word-level diff mode highlights the exact words that changed within a modified line. Three view modes: Edit (two textareas), Side by side, and Unified (like git diff).
When to use it
✓Comparing two versions of a document
✓Reviewing config changes before deploying
✓Checking what changed in an email draft
✓Spotting subtle differences in two code snippets
How to use
1Paste original text in the left panel (A)
2Paste modified text in the right panel (B)
3Switch to Side by side or Unified view in the toolbar
4Enable Word-level diff to highlight exact words changed within a line
5Click ⎘ Copy diff to copy a patch-format summary
💡Enable Ignore whitespace when comparing code that was reformatted — it won't flag indentation changes as differences.
↔Click ⇄ Swap to flip A and B — useful for reviewing the diff from the other direction.
📂Click Load sample to instantly see a realistic API docs diff — great for understanding all three view modes.

LinkedIn Post Formatter
Write and format LinkedIn posts with unicode styles — bold, italic, script, monospace and more. Live preview, one-click copy.
Open tool →
What it does
Converts plain text into unicode-styled text that LinkedIn renders as bold, italic, script and other styles in the feed. Select any text in the editor and click a style button to apply it. The right panel shows a live preview of exactly what your followers will see. One click copies the formatted post ready to paste into LinkedIn.
When to use it
✓Writing LinkedIn posts that stand out in the feed
✓Highlighting key lines or takeaways in bold
✓Adding visual structure without images
✓Making announcements or personal stories more readable
✓Formatting thought leadership posts for maximum scroll-stop
How to use

2Select the text you want to style — a heading, a key line, a word
3Click a style button in the toolbar — Bold, Italic, Script, Mono etc.
4See the result instantly in the Live Preview on the right
